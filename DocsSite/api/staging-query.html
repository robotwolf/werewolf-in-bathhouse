Staging API

Ask Rooms What They Mean

The query surface is the practical heart of Staging: callers ask for rooms, markers, connectors, semantic roles, and graph-relative candidates without scraping geometry or teaching Gideon a second job.

Ownership line: Staging explains published room truth. It does not choose topology, build shells, or quietly become runtime policy in a fake moustache.

Main Entry Points

PickBestRoom

Scores room, activity, marker, connector, semantic-intent, and graph-distance signals into one room selection.

PickBestRoomFromGenerator

Convenience wrapper that queries directly from a generator's spawned-room set.

PickBestMarkerInRoom

Chooses the best marker in a room by family and marker-tag clauses.

PickBestConnectorInRoom

Chooses the best connector in a room by occupancy, contract semantics, and optional compatibility with a reference seam.

Query Vocabulary

FStagingTagQuery

Required, blocked, and preferred gameplay tags with all-required vs any-required control.

FStagingMarkerQuery

Optional family restriction plus marker-tag filtering and scoring.

FStagingConnectorQuery

Connector occupancy and contract clauses: ConnectionType, PassageKind, BoundaryKind, ClearanceClass, ContractTag, and reference-connector compatibility.

FStagingRoomQuery

Room and activity tags, optional marker and connector clauses, semantic intent, origin-room graph clauses, and scoring weights.

Semantic Intent

Entry

Prefers Room.Function.Entry and Gideon arrival/booth/parking markers, then falls back to legacy entry-like naming only if needed.

Exit

Prefers Room.Function.Exit and Gideon.Exit, with entry-like fallback only when no dedicated exit exists.

Hide

Prefers Gideon.Hide, NPC.Activity.Hide, and hide-friendly room semantics like changing, maintenance, storage, and staff.

Connector-Aware Clauses

The connector query layer is how Staging starts reasoning about traversal contracts instead of just rooms and markers.

  • Require or prefer open connectors.
  • Require or prefer already connected connectors.
  • Filter by connection type, passage kind, boundary kind, clearance class, and contract tag.
  • Require compatibility with a reference connector when a caller is evaluating a handoff seam.

That lets callers ask for things like: a room with an open exterior seam, a room with a transition connector, or a room whose connector contract is compatible with a known traversal need.

Usage Shapes

Nearest Hide Room

Set semantic intent to Hide, supply an origin room, require reachability, and prefer nearest.

Activity-Matched NPC Room

Enable marker querying, require an NPC marker, and prefer the current activity tag across both room activity tags and marker tags.

Traversal-Compatible Room

Enable connector querying, require a matching connector, and use a reference connector to test compatibility against real traversal contracts.

Contained-exterior reminder: a room may look outside and still publish ordinary room and connector truth. Query semantics must honor the published contract, not vibes about weather.
