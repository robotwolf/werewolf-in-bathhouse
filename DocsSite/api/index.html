Who Owns Which Mess

Use this page when you need to know which subsystem should change and which one should be left alone with its coffee.

Ginny

Owns layout, room selection, hall approaches, required-path logic, and high-level generator policy.

Does not own: final shell art, runtime NPC orchestration, or pretending a room’s visual trickery changes topology.

Likely touchpoints: ARoomGenerator, UGinnyLayoutProfile, UGinnyRoomProfile

Mason

Owns stock and parametric construction. He builds shells when invited and can stay attached but idle for hero authored rooms.

Does not own: marker semantics, runtime crowd behavior, or your lovingly hand-built entry facade.

Likely touchpoints: UMasonBuilderComponent, UMasonConstructionProfile, room stock/parametric settings

Stagehand

Owns room truth publication and editor/runtime query seams: room signage, debug visualizers, connector semantics, marker selection helpers.

Does not own: layout policy or Gideon’s runtime crowd decisions.

Likely touchpoints: ARoomModuleBase, URoomSignageComponent, UStagehandDebugVisualizerComponent, URoomGameplayMarkerLibrary

Gideon

Owns runtime orchestration on top of generated space: arrivals, admission, POIs, crowd behavior, and related live management logic.

Does not own: room placement or connector truth. He consumes published room truth like a civilized runtime layer.

Likely touchpoints: AGideonDirector, GideonAdmissionBooth, runtime marker tags and queue markers

Open Gideon runtime doc

Butch

Owns dressing and decorative pass behavior. He makes the place look inhabited instead of merely legal.

Flo

Future configuration and flow orchestration layer. Useful to remember, not yet the boss of this page’s immediate problems.

Contained exterior pattern reminder: “outside by presentation, normal room by systems truth.”
